# Molecularity — META-ARCHITECTURE

> **Статус:** канон по мета-слою (Unity-клиент). Дополняет, а не отменяет
> [`GAME-CORE.md`](GAME-CORE.md) — тот остаётся единственным источником правды
> по правилам **игровой сессии**. Этот документ — источник правды по всему, что
> вокруг сессии: лобби, прогресс, валюта, магазин, квесты, окна, звук, сейв.
>
> Дата фиксации решений: 2026-06-23 (диалог-согласование с пользователем).

## 0. Граница Core ↔ Meta (повторение канона)

`Molecularity.Core` знает **только про одну игровую сессию** — один уровень +
инвентарь предметов на него. Всё остальное — мета и живёт в Unity:

| В Core (готово)                         | В Meta (этот документ, строим)                |
|-----------------------------------------|-----------------------------------------------|
| Граф молекул, правила хода, win/lose/stuck | Лобби-картинка, прогресс по главам         |
| Типы молекул, абилки, пассивки          | Кошелёк, магазин, IAP                          |
| `TurnResult` + события                  | Дейли-квесты                                   |
| Предметы уровня (как механика)          | Реклама, ревайв                                |
| JSON-уровни + солвер                     | Окна/попапы, звук, настройки, сейв            |

Core отдаёт меты только через **плоские DTO + события** (`TurnResult.Events`).
Unity не видит `Molecule`/`MoleculeGraph` напрямую — между ними фасад
(см. отдельный план интеграции ядра: фасад + DTO + layout + `ILevelSource`).

## 1. Зафиксированные решения

| Тема | Решение |
|------|---------|
| **Подход** | Гибрид: готовые SDK для рекламы/IAP/DI; Window-менеджер, квесты, кошелёк, магазин, save — свои, переносимые |
| **UI** | uGUI (prefab-попапы) |
| **Монетизация** | Сейчас — **заглушки за интерфейсами** (`IAdsService`, `IIapService`), реальный SDK позже |
| **DI** | VContainer |
| **Save** | JSON-файл в `persistentDataPath`, версионируемый (миграции). Заложить **флаг шифрования** в API, но шифрование пока не реализовывать |
| **Контент меты** | JSON (та же схема-подход, что `content/levels`; единый пайплайн, в будущем Google Sheets→JSON) |
| **Упаковка переносимых модулей** | **Отдельный git-репо** generic-модулей, подключение по git-url (UPM) |
| **Первый вертикальный срез** | Играбельный уровень (Bootstrap + Window-менеджер + Level Presenter по событиям + экран результата). Без экономики |
| **Прогрессия** | Миры/главы. Глава = скрытая картинка-силуэт (напр. ракета); на узлах силуэта — точки входа в уровни; прохождение снимает маску шейдером с куска картинки; собрал картинку = прошёл главу → следующая. **Это поле и есть лобби/главное меню** |
| **Точка ↔ уровень ↔ кусок** | 1 точка входа = 1 уровень = 1 фрагмент маски (маппинг 1:1) |
| **Жизни/энергия** | Нет. Свободная игра |
| **Доход монет** | Дейли-квесты + стартовый баланс + паки IAP. **НЕ** за прохождение (награда за уровень = кусок картинки) и **НЕ** rewarded |
| **Дейли-квесты** | 3 в день из пула, сброс раз в сутки |
| **Магазин** | Предметы уровня + Отключение рекламы + Паки монет (IAP) + Косметика (в будущем) |
| **Fail** | Попап: ревайв (= `PlusOneAll`, по канону) за rewarded-заглушку/монеты ИЛИ рестарт |
| **Настройки** | Звук/музыка/вибро + язык + ссылки (privacy/support) + сброс прогресса |

## 2. Карта модулей

Деление по принципу **переносимости**: generic-движки уезжают в отдельный
git-репо и переиспользуются в любом проекте; game-specific остаётся здесь и
питается данными.

### Слой 0 — Фундамент (generic, отдельный репо)
| Модуль | Ответственность | Тип |
|--------|-----------------|-----|
| **App Bootstrap (VContainer)** | Composition root: поднимает сервисы, регистрирует зависимости, запускает flow | generic |
| **Event Bus / Signals** | Развязка модулей: `LevelCompleted`, `CoinsChanged`, `ItemPurchased`, `ChapterCompleted` | generic |
| **Save / Persistence** | Один `PlayerData` ⇄ JSON в `persistentDataPath`, версия + миграции, флаг шифрования (хук есть, реализации нет) | generic |
| **Window Manager (uGUI)** | Стек экранов + попапы: open/close, слои, модальность/затемнение, Back, переходы | generic |

### Слой 1 — Сервисы (generic, отдельный репо)
| Модуль | Ответственность | Тип |
|--------|-----------------|-----|
| **Audio** | SFX (пул), музыка, ducking, mute/громкость (через Save) | generic |
| **Localization** | Таблицы строк, смена языка в рантайме | generic |
| **Ads** | `IAdsService` (interstitial + rewarded) — **заглушка** сейчас | generic + stub |
| **IAP** | `IIapService` (паки монет, remove-ads) — **заглушка** сейчас | generic + stub |
| **Analytics** | `IAnalytics`, подписка на Event Bus | generic |

### Слой 2 — Мета-геймплей (движок generic, контент — данные проекта)
| Модуль | Ответственность | Тип |
|--------|-----------------|-----|
| **Wallet / Currency** | Баланс монет, earn/spend, транзакции + события | движок generic |
| **Meta Inventory** | Какими предметами уровня владеет игрок; выдача/списание; **мост: собирает `PlayerInventory` для сессии Core** | мост (game-specific тонкий слой) |
| **Shop** | Каталог (JSON) + flow покупки; завязан на Wallet + IAP | движок generic, каталог-данные |
| **Daily Quests** | Определения (JSON) + трекинг через Event Bus + сброс в сутки + выдача наград | движок generic, квесты-данные |

### Слой 3 — Склейка и контент Molecularity (game-specific)
| Модуль | Ответственность |
|--------|-----------------|
| **Game Flow / Scene Loader** | splash → лобби-картинка → уровень → результат; на входе в уровень собирает сессию Core + инвентарь из меты |
| **Chapter / Picture Board (лобби)** | Силуэт-картинка, точки входа, раскрытие маски шейдером, прогресс по главам. Одновременно главное меню |
| **Level Presenter** | Потребляет `TurnResult.Events`, анимирует (`ValueChanged/Removed/Revealed/Spawned`). Стыкуется с фасадом ядра |
| **Settings Screen** | Звук/музыка/вибро/язык/ссылки/сброс |

## 3. Лобби как картинка-глава (центральная механика)

Глава = ассет-картинка (силуэт, напр. ракета) + маска + набор **точек входа**,
расставленных по узлам силуэта.

- **Маппинг 1:1:** одна точка входа → один уровень (`content/levels/level_X.json`)
  → один фрагмент маски. N точек в главе = N фрагментов.
- Прохождение уровня помечает точку пройденной и **снимает фрагмент маски шейдером**
  (reveal-эффект). Прогресс главы = доля снятых фрагментов.
- Снят весь силуэт (все точки пройдены) → `ChapterCompleted` → переход к следующей
  главе (новая картинка). Это же поле-картинка является **главным меню/лобби** —
  отдельного экрана меню нет, кнопки (магазин, настройки, квесты) — оверлеи поверх него.

**Данные главы (JSON, game-specific):**
```jsonc
{
  "chapterId": 1,
  "pictureAsset": "rocket",          // ссылка на спрайт/маску
  "entryPoints": [
    { "levelId": 1, "maskFragmentId": "frag_0", "anchor": [0.21, 0.74] },
    { "levelId": 2, "maskFragmentId": "frag_1", "anchor": [0.38, 0.55] }
    // ...
  ]
}
```
`anchor` — нормированная позиция точки на картинке (Unity сам позиционирует UI).
`levelId` ссылается на уровень из общего `content/levels`.

> Открытость следующей точки: по умолчанию линейно (точка `k+1` открывается
> прохождением `k`). Это деталь дизайна главы, фиксируется при наполнении контента.

## 4. Экономика

**Валюта:** золотые монеты (одна, мягкая).

**Источники дохода (только эти):**
1. **Дейли-квесты** — награда за выполнение.
2. **Стартовый баланс** — выдаётся при первом запуске.
3. **Паки монет (IAP)** — за реальные деньги (сейчас заглушка).

> Прохождение уровня монет **не даёт** — награда за уровень это кусок картинки.
> Rewarded-монеты в источниках сейчас не используются (rewarded зарезервирован под ревайв).

**Расход (магазин):**
- Предметы уровня: `RevealAll`, `PlusOneAll`, `Freeze`, `ChainBreak`, `Undo` — за монеты, падают в Meta Inventory.
- Отключение рекламы (remove-ads) — флаг (по канону через IAP).
- Паки монет — витрина IAP (заглушка-покупка).
- Косметика (скины молекул/фоны/палитры) — **в будущем**, отдельная витрина.

---

## 5. Дейли-квесты

- **3 квеста в день**, выбираются из пула, **сброс раз в сутки** (по локальному времени; время сброса фиксируется в Save).
- Прогресс трекается **через Event Bus** (квест-движок слушает `LevelCompleted`, `ItemUsed` и т.п. — никакой связи с Core напрямую).
- Награда — монеты; claim вручную из попапа квестов.
- Движок generic; конкретные определения — JSON-данные:
```jsonc
{ "questId": "win_3_levels", "type": "CompleteLevels", "target": 3, "reward": 50 }
```

---

## 6. Fail / Revive

При проигрыше (`GameStatus.Lose`) — попап:
- **Ревайв** = применить `PlusOneAll` к текущей сессии (по канону ревайв = `PlusOneAll`)
  и продолжить. Цена: rewarded-реклама (заглушка) ИЛИ монеты.
- **Рестарт** — перезапуск уровня с нуля.

Ревайв не создаёт новой сессии Core — это вызов предмета на текущей сессии.

---

## 7. Сейв

- Единый сериализуемый `PlayerData`: кошелёк, открытые главы/точки + снятые фрагменты,
  состояние и время сброса квестов, мета-инвентарь, настройки (звук/музыка/вибро/язык),
  флаг remove-ads.
- JSON в `persistentDataPath`, **версионируемый** (поле `version` + миграции при загрузке).
- Шифрование: API сериализатора принимает флаг/стратегию (`ISaveCodec`), но реальная
  реализация — **заглушка-passthrough** сейчас (закладка на будущее, не делаем).

## 8. Стратегия переносимости

1. **Отдельный git-репо** для всех generic-модулей (Слой 0–1 + движки Wallet/Shop/Quests).
   Подключение в проект по git-url через UPM. Каждый модуль — пакет с `package.json` + `asmdef`.
2. **Ноль ссылок на Molecularity** в generic-модулях: общение только через интерфейсы +
   Event Bus; весь контент — данные (JSON/ScriptableObject), а не код.
3. **Заглушки за интерфейсами** для платформенных SDK (`IAdsService`, `IIapService`,
   `IAnalytics`) — проект собирается и играется без реальных SDK; SDK подключается адаптером.
4. Game-specific остаётся в проекте: лобби-картинка, Level Presenter, мост-инвентарь,
   Scene Loader и данные (главы/квесты/товары/уровни).

> Подводный камень монорепо (из канона): при новой папке верхнего уровня под .NET-проект
> добавлять исключение в `.gitignore` (`!<dir>/**/*.csproj`), иначе CI падает с MSB1009.
> Для Unity-репо и отдельного git-репо модулей это правило тоже держать в уме.

## 9. Порядок реализации

1. **Фундамент:** Bootstrap (VContainer) → Event Bus → Save → Window Manager.
2. **Первый срез (цель):** Level Presenter (анимация по `TurnResult.Events`) + экран
   результата → *играбельный уровень в Unity*. Доказывает, что Core играется в движке.
3. **Мост в Core:** Meta Inventory → сборка `PlusOneAll`/предметов в `PlayerInventory` сессии.
4. **Лобби:** Chapter / Picture Board (силуэт + точки + reveal-шейдер) как главное меню.
5. **Экономика:** Wallet → Shop → IAP (заглушка).
6. **Реклама:** Ads (заглушка) → ревайв в Fail-попапе.
7. **Дейли-квесты:** движок + 3/день + сброс.
8. **Сервисы:** Audio, Localization, Analytics, экран настроек (можно параллельно с 5–7).
